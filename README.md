# Автоматизация отчетности о метриках приложения, объединяющего ленту новостей и мессенджер

### Описание проекта
Цель проекта - обеспечить прозрачную и регулярную коммуникацию с заинтересованными сторонами через автоматизированную доставку аналитических отчетов. Отчеты должны отправляться в Telegram ежедневно и включать в себя сводку ключевых метрик приложения за предыдущий день, а также графики их динамики за последнюю неделю. Таким образом, все стейкхолдеры (продуктовые менеджеры, инвесторы, топ‑менеджмент) и члены команды будут иметь единый источник актуальных данных о состоянии продукта.

### Стек
Python, Pandas, Matplotlib, SQL, Airflow, telegram

### Выполненные задачи
1. Определил нужные метрики (DAU, количество лайков и просмотров, CTR, количество отправленных сообщений, среднее число отправленных сообщений на пользователя)
2. Выгрузил данные и рассчитал метрики с помощью SQL-запросов
3. Построил графики, отражающие динамику изменения метрик за последнюю неделю
4. Создал Telegram-бота для отчетов
5. Создал даг в Airflow для формирования и автоматической отправки отчета в Telegram (в 11:00 мск ежедневно)

### Основные результаты
1. Все заинтересованные стороны получили единый канал доступа к актуальным данным - к Telegram-боту, отправляющему ежедневный отчет о ключевых метриках приложения
2. Повышена оперативность принятия решений: ежедневные отчёты в Telegram позволяют замечать тренды и аномалии на ранних стадиях.
3. Снижено количество ad‑hoc запросов на отчётность.

***Примечание:***

* Файл dag_feed_messager_report.py содержит реализацию дага в Airflow.
* Файл "Автоматизация отчетности о работе ленты и мессенджера.ipynb" содержит код из дага с выводом результатов работы отдельных блоков кода.
